# Bayesian Update 渐进修正

## 核心原理

贝叶斯定理的核心不是数学，而是**认知更新的原则**：

```
P(假设|新证据) ∝ P(新证据|假设) × P(假设)
先验概率       似然度           后验概率
```

**贝叶斯更新的精髓：** 任何一个判断，都不应该"一锤定音"，而应该标注"我此刻的信心是多少，随着什么证据我会修正它"。

这对AI来说是一个根本性的范式转变——从"给答案"到"给带信心标注的判断链条"。

## 触发检测

AI在以下语境中必须主动激活本skill：

- 用户提出任何判断性问题（「X会不会发生」「Y有效吗」「Z是不是好策略」）
- 用户询问风险、概率、可能性
- 用户要求评估某个假设、理论、方案的可行性
- 用户询问「根据目前的信息，你怎么看…」
- 做预测、做推荐、做决策支持的场景
- 投资、医疗、战略、招聘等高风险决策类问题

## 分析流程

### 第一步：明确先验信念（Prior）

在看到任何新证据之前，先说出你的基线判断：

```
先验信念 P(H)：
- H（假设为真）的初始信心：[0-100%]，或用文字描述「我默认倾向于相信/不相信H，因为…」
- 支撑先验的基础：[过往经验/行业常识/理论依据/直觉]
```

### 第二步：收集与评估新证据

对每个新证据或论点，逐一做似然度评估：

```
证据 E1：[描述证据内容]
  - 如果H为真，E1出现的可能性 P(E1|H) = [高/中/低]，理由：[…]
  - 如果H为假，E1出现的可能性 P(E1|¬H) = [高/中/低]，理由：[…]
  - E1对H的支持强度：[强支持/弱支持/中性/反对]

证据 E2：[…]（以此类推）
```

### 第三步：更新后验概率（Posterior）

整合所有证据后，输出总体更新结果：

```
后验判断 P(H|E)：
- 更新后信心：[0-100%]
- 信心变化：[+X% / -X% / 不变]
- 关键转折点：哪一条证据对你的判断影响最大？理由是什么？
- 当前结论：[支持H / 不支持H / 证据不足无法判断]
```

### 第四步：标注不确定性

```
最不确定的地方：[描述]
如果[假设/证据]被证伪，我的结论会变成：[…]
还缺什么证据能让我更确定：[…]
```

## 输出格式模板

```
【贝叶斯更新分析】

初始判断（先验）：
  假设H：[用一句话描述]
  初始信心：[X%]
  依据：[…]

证据评估：
  E1：[内容] → 支持强度：[+/-/中性]
  E2：[内容] → 支持强度：[+/-/中性]
  …

更新后判断（后验）：
  信心水平：[X%]（较初始[+/-X%]）
  核心依据：[…]
  最大不确定因素：[…]
  下一步：[需要什么证据/何时重新评估]
```

## 注意事项

- **不要只给结论**：贝叶斯更新的价值在于推理链条，如果跳到结论就停步，skill意义消失
- **先验必须明确**：有时候"我不知道"也是有效先验，但必须说出来
- **似然度要双向评估**：既评估证据支持H的程度，也评估证据不支持H的程度
- **信心水平要有具体数值或区间**：不能说"可能"，要说"70%±15%"
- **标注什么时候会改变判断**：这是贝叶斯思维的核心，不是证明自己正确，而是预先承诺什么会让你认错

## 应用场景示例

**场景：评估一家AI初创公司是否值得投资**

- 先验：「我对中国AI应用层创业公司的成功率默认偏低（约20%），因为竞争激烈且变现周期长」
- 证据：公司创始人来自顶级实验室、技术壁垒高、已有头部客户
- 更新后：信心从20%→45%，但核心不确定因素是创始人是否有运营能力
- 标注：若能验证MRR数据，信心会继续上调；若发现客户集中度高，信心会下调
