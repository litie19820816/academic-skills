# Grounded Theory 涌现式编码

## 核心原理

扎根理论（Grounded Theory）由Glaser和Strauss提出，核心思想是：**理论应该从材料中涌现，而非从预设假设中演绎。**

传统分析方法的毛病：先有结论，再找证据——这叫"验证性偏见"。扎根理论反其道而行：**不知道该发现什么，所以让材料自己说话。**

三阶段编码流程：

```
第一步：开放编码（Open Coding）
  → 对原始材料逐行/逐句标注，提取初始概念
  → 不预设类别，让类别从材料中自然浮现

第二步：轴心编码（Axial Coding）
  → 找出概念之间的关系结构
  → 将概念归类到更高级的范畴

第三步：选择性编码（Selective Coding）
  → 收敛到一个核心范畴
  → 用核心范畴串联起整个理论框架
```

## 触发检测

AI在以下语境中必须主动激活本skill：

- 用户提供原始材料（访谈、评论、反馈、文档）并要求分析
- 用户询问"这些材料说明了什么""能得出什么结论"
- 用户要求从大量非结构化文本中提取洞察
- 用户做竞品分析、用户研究、市场调研时提供原始数据
- 用户询问"这个领域/行业/群体有什么规律"
- 任何需要从定性材料生成理论的场景

## 重要原则

```
警告：不要直接给结论！
传统方法："这些材料说明X"（预设结论→找证据）
扎根理论："材料中浮现出X倾向，需要进一步验证"（材料→编码→涌现→理论）

过早归纳是本skill最大的敌人。
每一步都要问自己："我是在描述材料，还是在说我认为材料在说什么？"
```

## 分析流程

### 第一步：开放编码（Open Coding）

**任务：** 对原始材料逐行/逐句/逐事件标注，提取初始概念。

**操作方式：**

```
对每一段材料，问：
- 这里发生了什么？（描述）
- 这个现象的关键特征是什么？（概念）
- 这个概念与其他材料中的什么概念相似或不同？（比较）

每个概念用 [概念名称] 标注，后面附简短定义。
```

**输出格式：**

```
【开放编码结果】

材料片段 A1：[原文摘录]
  初始概念：C1=[概念名称] - [简短定义]
  材料片段 A2：[原文摘录]
  初始概念：C2=[概念名称] - [简短定义]
  …

涌现出的初始概念清单（共N个）：
  C1：[名称] - [定义] - [出现的材料片段数]
  C2：[名称] - [定义] - [出现的材料片段数]
  …
```

**原则：**

- 概念要接近材料，不要过度抽象化
- 一个材料片段可以标注多个概念
- 相似概念要合并，差异概念要区分
- 暂不下结论，只是记录

### 第二步：轴心编码（Axial Coding）

**任务：** 找出概念之间的关系结构，将概念归类到更高级范畴。

**操作方式：**

```
每个概念问：
- 这个概念的前因是什么？（什么导致了这个现象）
- 这个概念的后果是什么？（这个现象会导致什么）
- 这个概念与哪些其他概念相关？在什么条件下相关？
- 这个概念的本质是什么类别的现象？（因果/背景/条件/策略/结果）
```

**关系类型：**

| 关系类型 | 含义 | 示例 |
|---------|------|------|
| 因果关系 | X导致Y | 焦虑→寺庙消费 |
| 从属关系 | X是Y的条件 | 高学历→高收入 |
| 对应关系 | X对应于Y | 内卷→躺平 |
| 阶段关系 | X是Y的前置阶段 | 试用→转化 |

**输出格式：**

```
【轴心编码结果】

范畴 A：[名称]
  包含概念：[C1, C3, C7]
  范畴性质：[这个范畴代表什么类型的社会现象]
  关系结构：[与其他范畴的连接方式]

范畴 B：[名称]
  …

核心范畴候选：[列出1-3个最核心的范畴，附理由]
```

### 第三步：选择性编码（Selective Coding）

**任务：** 从轴心编码中收敛到一个核心范畴，用核心范畴串联起整个理论框架。

**操作方式：**

```
选择核心范畴的标准：
- 它是否在所有/大多数主要范畴中处于中心位置？
- 它是否能解释大部分概念之间的关系？
- 它是否足够抽象但仍有经验基础？

围绕核心范畴叙事：
- 核心范畴是什么？（用一句话描述）
- 它如何串联起所有其他范畴？（叙述故事线）
- 核心范畴的前因、后果、边界条件是什么？
```

**输出格式：**

```
【选择性编码结果】

核心范畴：[名称]
  定义：[用学术语言定义]
  故事线：[核心范畴如何串联起整个分析]
  前因：[什么条件下核心范畴出现/增强]
  后果：[核心范畴出现后会导致什么]
  边界条件：[在什么情况下理论不适用]

涌现理论框架：
  [用结构化的方式描述这个自下而上生成的理论]

理论贡献：
  [这个扎根分析对理解该现象的独特贡献是什么]
```

## 完整输出模板

```
【扎根理论分析】

原始材料概述：
  材料类型：[访谈/评论/文献/…]
  材料量：[N条/ N字]
  来源：[描述]

第一步：开放编码
  [输出所有初始概念清单]

第二步：轴心编码
  [输出范畴及其关系结构]

第三步：选择性编码
  [输出核心范畴和理论框架]

方法论说明：
  资料饱和度：[现有材料是否足够支撑理论？缺什么？]
  理论敏感性：[分析者是否已充分理解材料语境？]
  涌现vs预设：[核心范畴是否真正从材料中涌现，而非预设？]

局限性：
  [本研究未能触及的维度，可能的盲点]
```

## 质量检验清单

在完成分析后，自问：

- [ ] 我是否给每个重要材料片段都做了编码？
- [ ] 我的概念是否足够接近材料，还是过度抽象？
- [ ] 我有没有强行把材料塞进预设的框架？（如果发现，要重做）
- [ ] 核心范畴是否真正具有统领性，还是强扭的？
- [ ] 我的理论框架是否能够解释现有材料的主要部分？
- [ ] 是否存在反驳核心范畴的材料？（如果有，需要正视而非忽略）

## 注意事项

- **材料量要足够**：扎根理论需要足够的材料量（通常10-20个以上独立片段）才能涌现理论
- **保持悬置**：分析者要不断检视自己是否在"验证"预设想法，而非"发现"新材料的意思
- **允许涌现vs验证**：如果分析过程中发现材料不支持你的预设假设，这是好事，说明理论更真实
- **标注引用**：每个概念都要能追溯到原始材料，保持可审计性
- **不止一个核心范畴**：复杂现象可能有2-3个核心范畴并列，不要强行合并
