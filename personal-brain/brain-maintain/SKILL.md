# Maintain — 知识脑维护

定期执行知识脑健康检查，强制执行质量和完整性标准。

---

## 维护协议（每周一次）

### 1. 反向链接检查（Iron Law Enforcement）

**目标：** 找到所有"悬挂提及"——在文中提到某个实体，但没有在该实体的档案页创建反向链接。

```
对每个包含 [[ 或脑内链接的页面：
  提取所有被链接的实体 slug
  对每个实体：
    打开实体档案
    检查是否有反向链接指向当前页面
    如果没有 → 标记为"缺失反向链接"
```

**修复：** 为每个缺失的反向链接，在实体档案的 Timeline 或 See Also 部分添加条目。

### 2. 引用完整性审计

**目标：** 确保所有事实陈述都有 `[Source: ...]` 引用。

```
对脑中的每个 .md 文件：
  扫描所有非引用行（排除 frontmatter、链接、标题）
  如果发现缺乏 [Source: 的事实陈述 → 标记
```

**来源权威性优先级验证：**
1. 用户直接陈述（最高）
2. 综合真相
3. 时间线条目
4. 外部来源（最低）

### 3. 归档规则验证

**目标：** 确保内容在正确目录。

| 问题模式 | 修复 |
|----------|------|
| 分析某主题的页面在 `sources/` | 移到适当主题目录 |
| 关于某人的页面在 `sources/` | 移到 `people/` |
| 来自会议的关于某公司的信息只在 `meetings/` | 同时更新 `companies/` |
| 可复用框架在 `sources/` | 移到 `concepts/` |

### 4. 过时信息检测

**目标：** 标记可能过时的页面。

- 上次更新 > 3 个月前的页面，且有 `[STALE]` 标记
- 包含时间敏感信息（项目状态、联系方式）但超过 6 个月未更新的页面
- 标记为 `[Needs review]` 并追加到维护队列

### 5. 孤立页面检测

**目标：** 找到没有被任何页面链接的页面（孤儿）。

```
从脑目录中的每个 .md 文件：
  提取所有内部链接
  构建链接图
  找没有任何入站链接的页面
```

**处理：**
- 如果页面有价值但孤立 → 添加到相关页面的 See Also
- 如果页面确实无价值 → 归档或删除

### 6. 搜索质量基准

**目标：** 确保知识脑可被可靠搜索。

- 抽样 10 个已知实体，测试能否通过名字、别名、部分匹配找到
- 标记搜索不到的已知实体
- 检查是否有 slug 格式不一致问题

## 执行报告

维护完成后，生成结构化报告：

```
# 知识脑维护报告 — YYYY-MM-DD

## 反向链接
- 检查页面数：N
- 缺失反向链接：N（已修复 N）
- 悬挂提及：N

## 引用完整性
- 缺少引用的事实：N
- 来源冲突已标注：N

## 归档
- 错误归档页面：N（已修复 N）
- 新孤立页面：N

## 过时内容
- 需审核的页面：N
- 已标注 [STALE]：N

## 搜索质量
- 抽样通过率：X/Y
- 失败条目：N

## 总体评分
[ ] 优秀 — 无问题
[ ] 良好 — 微小问题已修复
[ ] 需要关注 — 若干问题需人工审核
```
