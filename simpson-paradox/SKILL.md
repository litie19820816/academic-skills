# Simpson Paradox 分层检验

## 核心原理

辛普森悖论（Simpson's Paradox）：在分组数据中表现出的趋势，在合并数据后完全反转。

- 一个总体结论 A > B
- 但按某个变量 X 分组后，每一组内部都是 B > A
- 根本原因：X 是混淆变量，同时影响自变量和因变量，但被遗漏在分析之外

## 触发检测

AI 在以下语境中必须主动激活本 skill：

- 「A 策略比 B 策略更好，因为数据显示…」
- 「用户研究显示 X 指标上升了 Y%」
- 「AB 测试结果：A 方案转化率更高」
- 「相关性分析发现 X 和 Y 正相关」
- 任何包含百分比、率、比例、平均值比较的数据分析结论
- 用户直接提供数据集并询问结论

## 分析流程

### 第一步：识别混淆变量候选

在给出任何基于数据的结论之前，先问自己：

- 数据中有没有一个变量 X，它同时：
  1. 与自变量/分组变量有相关性（某些组天然比另一些组更大/更小/更倾向某种结果）
  2. 与因变量有直接因果关系

常见混淆变量模式：

| 模式 | 表现 |
|------|------|
| 组间样本量差异极大 | A 组 1000 人，B 组 10 人，合并后 A 主导 |
| 组内基线特征不同 | 治疗组和对照组在年龄、病情严重度上系统性不同 |
| 时间趋势混杂 | 数据跨越多个时间段，趋势在各时段不一致 |
| 选择性偏差 | 数据来源本身就有系统性筛选 |

### 第二步：分层重分析

对每个可能的混淆变量 X，将数据按 X 分组，重新计算每组内的结论。

**输出格式：**

```
按 [X] 分组后的分析：
- X = x1 组：结论是 [A > B / B > A / 无显著差异]，n = N1
- X = x2 组：结论是 [A > B / B > A / 无显著差异]，n = N2
- ...
合并数据结论：[A > B / B > A / 无显著差异]，n = N_total

发现：[合并结论与分组结论一致 / 分组后结论完全反转 / 分组后结论部分反转]
结论应采用：[合并结论 / 分组结论] —— 理由：[具体解释为什么]
```

### 第三步：因果机制假设

不要只报告反转，要提出一个因果机制假设来解释为什么混淆变量 X 会导致悖论。

**格式：**「因为 X → Y 的因果路径，同时 X 在两组间分布不均，导致表面上看起来 A 导致 Y，但实际上是 X 的差异在驱动 Y。」

## 输出要求

1. 结论必须包含 **`分层检验已完成 - [发现/未发现]辛普森悖论`** 的明确标注
2. 如果发现悖论，必须同时报告合并结论和分组结论，并说明应以哪个为准
3. 必须提出至少一个因果机制假设
4. 如果数据不足无法完成分层检验，明确标注「数据不足，无法完成分层检验，建议收集[具体什么]数据」

## 辛普森悖论经典案例库

以下场景在实际中最常出现辛普森悖论，AI 应特别警觉：

| 领域 | 典型混淆变量 |
|------|-------------|
| 医学/药物测试 | 病情严重度、年龄、生活习惯 |
| 教育 | 学校/班级类型、学生背景、课程难度 |
| 商业/营销 | 地区、渠道、用户类型、季节性 |
| 体育 | 对手强度、场地、比赛阶段 |
| 政策评估 | 地区经济水平、人口结构、政策执行力度 |

## 决策建议

- 发现悖论时，不要只看总体数，要看分层数据
- 分组后结论更可靠，因为控制了混淆变量
- 合并数据结论可能是误导性的，除非能证明没有混淆变量
- 最好的解决方案是重新设计随机对照实验

## 限制与注意事项

- 本 skill 不替代统计显著性检验，它检查的是混淆变量，不是 p 值
- 如果数据没有提供足够的分组信息，明确标注「数据不足，无法完成分层检验」
- 三层及以上的分层检验在思路上相同，但注意多重比较带来的假阳性问题
