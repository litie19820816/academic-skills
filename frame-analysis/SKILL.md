# Frame Analysis 情境解码

## 核心原理

戈夫曼（Erving Goffman）的框架理论核心洞察：

```
框架 = 人们用来定义"此刻正在发生什么"的认知结构

同一个事件：
- 框架A：[用框架A来看] → 事件被解读为"技术故障" → 处理方式：道歉+维修
- 框架B：[用框架B来看] → 事件被解读为"道德失败" → 处理方式：追责+惩罚

框架决定认知，认知决定行动。
```

**本skill的核心任务：** 识别各方的框架，分析框架冲突，发现框架转换点。

## 触发检测

AI在以下语境中必须主动激活本skill：

- 舆情分析、品牌危机
- 谈判准备（预判对方如何定义这场谈判）
- 公关策略制定
- 政策沟通分析
- 跨文化沟通
- "为什么对方不承认这是问题"
- "为什么双方的沟通总是鸡同鸭讲"
- 任何涉及"看问题的角度"的讨论

## 分析流程

### 第一步：识别所有涉事方及其框架

```
涉事方框架清单：

【方A】使用的框架：[框架名称]
  - 这个框架的核心假设是什么？
  - 在这个框架下，"正当的"行为是什么？
  - 在这个框架下，对方应该承担什么责任？
  - 这个框架的关键词/叙事是什么？

【方B】使用的框架：[框架名称]
  [同上]

【方C】使用的框架：[框架名称]
  [同上]
```

### 第二步：分析框架之间的冲突点

```
框架冲突分析：

冲突点1：
  - 描述：[双方在什么具体问题上存在框架冲突？]
  - 方A框架下：[这个问题的性质是什么]
  - 方B框架下：[这个问题的性质是什么]
  - 冲突的本质：[框架假设的根本差异]

冲突点2：[同上]
```

### 第三步：找出框架的支撑/瓦解条件

```
框架稳定性分析：

【方A的框架】
  支撑条件：[什么会让这个框架更稳固？]
  瓦解条件：[什么会动摇这个框架？]
  框架转换触发点：[什么事件/证据/叙事会让方A切换框架？]

【方B的框架】
  [同上]
```

### 第四步：设计框架干预策略

```
框架干预策略：

方案A：框架转换
  - 策略：[引入什么新证据/叙事，可以把对方引向另一个框架？]
  - 风险：[对方可能如何抵抗？]
  - 时机：[什么时候抛出这个框架转换最有效？]

方案B：框架叠加
  - 策略：[能否找到一个双方框架都能容纳的更高层框架？]
  - 例如：从"谁对谁错"框架 → 切换到"共同解决问题"框架

方案C：框架揭示
  - 策略：[不改变框架，而是让对方意识到自己正在使用某个框架]
  - 作用：元认知层面的干预，让对话回到框架选择的层面
```

## 输出格式

```
【框架分析报告】

涉事方框架：
  [方A]：[框架名称] → 核心假设：[…] → 关键词：[…]
  [方B]：[框架名称] → 核心假设：[…] → 关键词：[…]

框架冲突地图：
  冲突点1：[…]（框架差异：[…]）
  冲突点2：[…]（框架差异：[…]）

框架转换机会：
  [最脆弱的框架 / 最容易说服的一方 / 推荐的框架转换路径]

策略建议：
  [针对每个冲突点的具体框架干预策略]
```

## 框架类型库

以下是常见的框架类型，分析时优先检查这些是否适用：

| 框架类型 | 核心逻辑 | 典型叙事 |
|---------|---------|---------|
| 道德框架 | 这是对/错问题 | "良心"、"责任"、"耻辱" |
| 技术框架 | 这是能力/效率问题 | "优化"、"改进"、"最佳实践" |
| 法律框架 | 这是否合规问题 | "违法"、"权益"、"合同" |
| 经济框架 | 成本/收益问题 | "投资回报"、"浪费"、"效率" |
| 安全框架 | 风险/威胁问题 | "隐患"、"危机"、"保障" |
| 身份框架 | 谁是"我们"/"他们" | "爱国"、"背叛"、"忠诚" |
| 叙事框架 | 这是一个什么故事 | "英雄"、"受害者"、"恶棍" |

## 注意事项

- **框架无对错**：每个框架都有其内在逻辑，不要假设自己的框架是"正确的"
- **框架决定感知**：同一个事件在不同框架下看起来完全不同
- **先识别再干预**：不先理解对方的框架，就无法有效改变对话方向
- **框架有文化差异**：不同文化背景的人默认框架系统不同
