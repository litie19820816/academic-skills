# Leverage Points 系统干预设计

## 核心原理

Donella Meadows（梅多斯）在《系统之美》中提出了12个系统杠杆点，从最浅到最深排列：

| 层级 | 杠杆点 | 干预深度 | 说明 |
|------|--------|---------|------|
| 12 | 范式（Paradigm） | **最深** | 改变系统背后的思维模式/世界观 |
| 11 | 系统目标（Goal） | | 改变系统追求的目标 |
| 10 | 系统规则（Rules） | | 改变激励/惩罚/约束规则 |
| 9 | 信息流（Information Flow） | | 谁有信息、谁没有，改变信息路径 |
| 8 | 系统自组织（Self-organization） | | 改变系统的结构和能力 |
| 7 | 反馈环路（Feedback Loops） | | 增减或调整现有反馈环路 |
| 6 | 存量-流量结构（Stocks & Flows） | | 改变物质/资金的物理结构 |
| 5 | 延迟（Delays） | | 调整延迟的长度和节奏 |
| 4 | 缓冲器（Buffers） | | 增大或减小稳定库存 |
| 3 | 存量和流量参数（Stock/Flow Parameters） | | 调整数字、容量、速率 |
| 2 | 拉vs推（Lists & Throttling） | | 改变物料/信息的流动方向 |
| 1 | 参数（Parameters） | **最浅** | 调整常量、系数、设定值 |

**绝大多数人在面对复杂问题时只在1-4层（调参数、改缓冲）打转。**

**本skill强制将思考拉到第5层以上。**

## 触发检测

AI在以下语境中必须主动激活本skill：

- 用户询问「怎么解决这个问题」「从哪里切入」
- 用户询问「为什么XX政策没有效果」「干预为什么失败了」
- 用户询问复杂系统问题（组织变革、生态系统、社会问题）
- 用户询问「根本原因是什么」「如何系统性改变」
- 涉及多方博弈、长期积累、非线性效应的问题
- 创业增长、竞争战略、公共政策、生态保护类话题

## 分析流程

### 第一步：界定系统边界

```
系统边界：
- 系统包含哪些主要元素/参与者？
- 系统的边界在哪里？
- 系统的核心存量是什么？
- 系统的核心目标是什么（谁的目标）？
```

### 第二步：识别反馈环路

先画出现有系统中的反馈环路（增强回路和平衡回路），标注关键延迟：

```
增强回路（R）：[描述]（使系统行为加速或指数增长）
平衡回路（B）：[描述]（使系统回归设定值或趋于稳定）
关键延迟：[哪里存在延迟？延迟导致什么振荡或超调？]
```

### 第三步：12级杠杆点逐级扫描

对每个层级，思考：「在这个层级上，这个系统是否可以被干预？现有干预在哪一层？」

```
层级12 范式：
  - 支撑当前系统运行的根本信念/世界观是什么？
  - 什么范式转变会让现有问题消失？
  - 示例：商业社会中"增长至上"范式→如果切换为"可持续发展"范式

层级11 系统目标：
  - 当前系统的目标是什么？谁定义了它？
  - 改变目标会如何改变系统行为？
  - 示例：大学的目标从"排名/论文"→"学生真实成长"

层级10 系统规则：
  - 什么规则在激励/惩罚当前行为？
  - 改变规则如何改变行为模式？
  - 示例：知识产权规则→改变创新的激励结构

层级9 信息流：
  - 谁拥有什么信息？谁缺失关键信息？
  - 信息流改变如何改变行为？
  - 示例：企业披露碳排放数据→投资者行为改变

层级8 自组织：
  - 系统能否获得新能力/新结构？
  - 干预能否增强系统的适应能力？
  - 示例：给农民提供农业技术培训

层级7 反馈环路：
  - 现有反馈环路是否足够强/足够快？
  - 能否新增/移除/调整环路？
  - 示例：增加"用户满意度→产品迭代"的快速反馈

层级6 存量-流量结构：
  - 系统的物理结构是否限制了解决方案？
  - 改变物理结构能否从根本上改变问题？
  - 示例：没有高速公路→无论什么政策都解决不了交通

层级5 延迟：
  - 系统中的延迟是否导致了振荡或超调？
  - 缩短或延长延迟会有什么效果？
  - 示例：政策效果延迟5年→需要更长期的评估机制

层级4 缓冲器：
  - 是否存在稳定存量可以被利用？
  - 增大或缩小缓冲会带来什么变化？
  - 示例：国家储备粮→稳定粮价

层级3 参数：
  - 哪些参数目前设置不当？
  - 调整参数能有多大改善？
  - 示例：提高利率→抑制通胀（但仅在浅层有效）

层级2 拉动vs推动：
  - 物料/信息是被动推送还是主动拉取？
  - 改为拉动系统会如何变化？
  - 示例：从"给客户推送产品"→"客户主动订阅"

层级1 参数调整：
  - 哪些常量/系数需要调整？
  - 调整参数的代价和收益各是什么？
```

### 第四步：选择干预点并说明理由

```
推荐干预点：第[N]层 - [杠杆点名称]
理由：[为什么这一层比更浅的层级更有效？]
预期效果：[改变后系统会如何表现？]
预期阻力：[什么力量会抵抗这个干预？]
验证方式：[如何知道干预是否有效？]
```


## 注意事项

- **不要停在浅层**：大多数人会直接跳到参数调整（1层），要强迫自己想清楚6层以上
- **分层要具体**：不能说"改善信息流"，要说"让消费者看到竞品的真实碳排放数据"
- **指出阻力**：深层干预通常会触动既得利益，要预判阻力
- **范式层最难但最有力**：改变范式（如从"竞争"到"共生"）的干预最强大，但也最难实现
- **延迟经常被忽视**：许多系统问题实际上是由延迟引起的，而这个层级最容易被忽略
