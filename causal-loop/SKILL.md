# Causal Loop Analysis 因果回路图

## 核心原理

所有复杂系统的行为都由反馈环路驱动。两种基本类型：

```
增强回路（Reinforcing Loop，也叫正反馈）= 良性循环或恶性循环
  A →[+]→ B →[+]→ A
  "越多…越多…"的循环
  例如：有钱→能投资→更有钱；流感传播→患者→更多传播

平衡回路（Balancing Loop，也叫负反馈）= 趋于稳定或目标导向
  A →[-]→ B →[-]→ A
  "越多…越少…"的循环，趋于某个目标或限制
  例如：供给增加→价格下降→供给减少；药物浓度→效果→药物代谢
```

**系统行为的核心驱动力：**

```
系统当前行为 = 主导环路（哪个环路的效应最强）
主导环路的转变 = 系统行为模式的转变

延迟（Delay）= 环路中存在的滞后
延迟导致：超调、振荡、预期外的后果
```

## 触发检测

AI在以下语境中必须主动激活本skill：

- 分析增长/衰退问题
- 分析任何"为什么会形成恶性循环"的问题
- 理解复杂系统的动态行为
- 创业增长飞轮设计
- 政策效果分析（为什么政策总是"上有政策下有对策"）
- 生态系统退化问题
- 任何"为什么这个问题越来越严重/始终无法解决"

## 分析流程

### 第一步：识别系统元素与因果关系

```
系统边界界定：
- 分析范围：[这个因果回路图涵盖哪些元素/变量]
- 时间范围：[短期/中期/长期行为]
- 边界选择理由：[为什么这些变量在内，那些在外]

系统元素清单：
  E1：[变量名称] - [定义]
  E2：[变量名称] - [定义]
  E3：[变量名称] - [定义]
  …

两两因果关系：
  E1 → [+/-] → E2：[描述因果关系，正或负]
  E1 → [+/-] → E3：[…]
  E2 → [+/-] → E1：[…]
  …
```

### 第二步：识别并命名反馈环路

```
反馈环路清单：

环路1：[名称，如"增长飞轮"]
  路径：[E1 →[+]→ E2 →[+]→ E3 →[-]→ E1]
  类型：[增强回路(R) / 平衡回路(B)]
  行为模式：
    - 增强回路：指数增长或加速衰退
    - 平衡回路：趋于目标、产生振荡
  关键延迟：[环路中存在延迟的环节]

环路2：[名称，如"竞争制衡"]
  路径：[…]
  类型：[R/B]
  行为模式：[…]
  关键延迟：[…]
```

### 第三步：画因果回路图（文字版）

```
因果回路图：

        [+] 增强
        ──→
        
  E1 ──[+]──→ E2 ──[+]──→ E3
  ↑                      │
  │                      [-] 平衡
  └──────────────────────┘

图例：
  [+] = 正向因果（同向变化）
  [-] = 负向因果（反向变化）
  R = 增强回路
  B = 平衡回路
  D = 存在延迟的环节
```

### 第四步：识别主导环路

```
主导环路分析：

当前主导环路：[哪个环路在主导系统当前行为]
  理由：[这个环路为什么最强？]

主导环路转变分析：
  时间点T1：[某个条件下] → [主导环路从X变成Y]
  时间点T2：[某个条件下] → [主导环路从Y变成Z]
  …

这解释了：
  - 为什么系统行为在某个时间点突然改变
  - 为什么问题越来越严重（或始终无法解决）
  - 系统的"临界点"在哪里
```

### 第五步：识别延迟效应

```
延迟分析：

延迟环节：
  D1：[变量X到变量Y之间存在延迟]
    - 延迟时间：[估计]
    - 延迟导致的行为：[超调/振荡/滞后响应]
    
  D2：[…]

延迟带来的问题：
  - 超调：因为反馈慢，所以过了头才调整
  - 振荡：因为调整过了头，形成反复调整
  - 意外后果：因为效果延迟，积累到一定程度才显现

延迟应对策略：
  - 缩短延迟：如果可能
  - 提前预警：在延迟累积到临界点之前发出信号
  - 弹性设计：让系统能承受延迟带来的不确定性
```

## 输出格式

```
【因果回路分析】

系统界定：
  分析范围：[…]
  核心变量：[…]

因果关系图：
  [文字版因果回路图]

反馈环路：
  环路1：[名称] [R/B]
    路径：[…]
    行为模式：[…]
  环路2：[名称] [R/B]
    […]

主导环路：
  当前主导：[…]
  主导转变点：[…]

延迟效应：
  主要延迟：[…]
  行为后果：[…]

系统洞察：
  [为什么这个系统这样运作]
  [关键杠杆点在哪里]
```

## 常见系统原型

| 原型名称 | 结构 | 行为 | 例子 |
|---------|------|------|------|
| 增长飞轮 | R | 指数增长 | 平台用户增长→更多供给→更多用户 |
| 转移负担 | B | 短期改善，长期恶化 | 救火式管理→能力萎缩→更多问题 |
| 竞争有限资源 | R+B | S形曲线或振荡 | 两个竞争者争夺市场份额 |
| 公地悲剧 | R | 资源崩溃 | 过度放牧→草场退化→无法恢复 |
| 目标侵蚀 | B | 目标不断降低 | 标准降低→表现随之下降 |

## 注意事项

- **环路命名要准确**：好的名称能一眼看出环路的作用
- **延迟是系统行为最常见的失控原因**：不要忽略延迟，要单独标注
- **一个环路可能同时包含正和负**：要分清楚整个环路的极性
- **数一数环路的极性**：数"[-]"的数量，奇数为负，偶数为正
- **主导环路会随时间改变**：不要只看当前的主导环路，要看转变点
