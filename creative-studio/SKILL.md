# 文学创作工作室 (Creative Studio)

> 一个专为AI Agent设计的专业文学创作协作系统，支持长篇小说、剧本、游戏叙事等多种文学形式的创作。

## 核心定位

文学创作工作室是一套结构化的创作方法论，将专业文学创作拆解为可协作的子任务，由不同专长的Agent协同完成。

## 适用场景

- **长篇小说创作**：史诗奇幻、科幻巨著、历史文学
- **剧本创作**：电影剧本、电视剧、游戏剧本
- **世界观构建**：游戏叙事、世界设定书
- **短篇创作**：中短篇小说、散文集
- **IP开发**：系列作品、跨媒体叙事

## 核心架构

### 三层协作体系

```
┌─────────────────────────────────────────────┐
│           Creative Director                  │
│         (创意总监 - 战略层)                  │
│   把握创作方向，解决核心冲突，最终拍板       │
└─────────────────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│ World       │ │ Character   │ │ Plot        │
│ Architect   │ │ Lead        │ │ Architect   │
│ (世界观架构) │ │ (人物主笔)  │ │ (情节架构)  │
└─────────────┘ └─────────────┘ └─────────────┘
        │             │             │
        └─────────────┼─────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│           执行层 (专精Agent)                 │
│  Scene Writer | Dialogue Specialist         │
│  Prose Polisher | Researcher                │
│  Continuity Tracker | Editorial Director    │
└─────────────────────────────────────────────┘
```

### 完整工作流示例

```
用户请求："帮我写一段秦琼大战尉迟恭的战斗场面"

Step 1: Creative Director 分析请求
├─ 识别核心目标：展示双方武艺、推进情节、塑造人物
├─ 确定风格基调：评书风、英雄气、戏剧张力
└─ 输出：创作目标文档（核心支柱、风格要求）

Step 2: Character Lead 调取人物档案
├─ 秦琼：锏法、性格、武艺特点
├─ 尉迟恭：枪法、性格、战斗风格
└─ 输出：人物战斗档案卡

Step 3: World Architect 确认背景
├─ 战斗地点：历史地理信息
├─ 时代背景：武备、马匹、甲胄
└─ 输出：场景背景说明

Step 4: Plot Architect 设计情节
├─ 战斗起因：为什么打
├─ 战斗结果：谁赢、怎么赢
├─ 戏剧张力：转折点、悬念
└─ 输出：场景情节大纲

Step 5: Scene Writer 执行写作
├─ 输入：以上所有材料
├─ 执行：撰写战斗场面初稿
└─ 输出：场景文本（初稿）

Step 6: Dialogue Specialist 优化对话
├─ 战斗中的喊话
├─ 心理活动描写
└─ 输出：对话优化版

Step 7: Prose Polisher 润色风格
├─ 评书式节奏调整
├─ 修辞强化（"说时迟，那时快"）
└─ 输出：终稿

Step 8: Continuity Tracker 检查
├─ 时间线是否一致
├─ 人物状态是否前后矛盾
└─ 输出：连贯性报告

Step 9: 返回用户
├─ 展示终稿
└─ 如有重大决策点，先确认再继续
```

## 协作规则

### 决策流程

```
Question → Options → Decision → Draft → Approval
```

1. **问题识别**：识别当前创作决策点
2. **方案提供**：列出可选方案及优劣
3. **决策拍板**：Creative Director 做出决定
4. **草案输出**：执行层Agent完成初稿
5. **审批确认**：质量层审核通过

### 升级机制

```
执行层 → 设计层 → 战略层 → 用户
```

- 执行层无法决断时，升级至设计层
- 设计层出现分歧时，升级至战略层
- 战略层无法解决时，提请用户决策

### 质量门禁

每个创作节点需通过：
1. **一致性检查**：Continuity Tracker 验证
2. **风格检查**：Prose Polisher 审核
3. **质量检查**：Editorial Director 终审

### 检查点与用户确认

#### 必须用户确认的场景

以下操作**必须暂停并请求用户确认**后才能继续：

| 场景 | 原因 | 确认方式 |
|------|------|----------|
| 故事核心方向变更 | 影响整体创作 | 展示变更前后对比，请求明确同意 |
| 主要人物命运决定 | 不可逆操作 | 提供2-3个选项，由用户选择 |
| 世界观重大设定修改 | 影响范围广 | 说明影响范围，请求确认 |
| 删除/大幅修改已完成内容 | 工作损失风险 | 说明删除原因和影响 |
| 风格基调调整 | 影响整体观感 | 提供示例对比 |
| 跨章节内容调整 | 连锁影响 | 说明连锁变化 |

#### 可自主执行的场景

以下操作Agent可**自主执行**，完成后汇报：

| 场景 | 条件 |
|------|------|
| 细节描写润色 | 不改变情节走向 |
| 人物对话优化 | 符合已确立性格 |
| 场景氛围增强 | 不改变核心内容 |
| 错别字/语法修正 | - |
| 格式统一 | - |

#### 高风险操作识别

Agent在执行前自动检查风险等级：

```
高风险 = 影响核心支柱 OR 影响多章节 OR 涉及用户明确偏好
中风险 = 影响单个章节 OR 涉及人物细节
低风险 = 仅涉及语言表达
```

### 异常处理与边界条件

#### 常见异常场景

| 异常场景 | 处理方式 |
|----------|----------|
| 用户需求模糊 | Creative Director 追问澄清，提供2-3个方向选项 |
| 设定冲突 | Continuity Tracker 标记 → 对应 Lead 修正 → Editorial Director 确认 |
| 创作卡住 | Scene Writer 返回问题 → Plot Architect 调整方案 |
| 时间线混乱 | Continuity Tracker 生成时间线图 → Character Lead 核对人物轨迹 |
| 风格不统一 | Prose Polisher 标注问题段落 → 统一风格指南 |

#### Fallback路径

当主流程失败时，按以下顺序降级：

```
1. 尝试恢复：回到上一个确认点重新执行
    ↓（失败）
2. 简化执行：减少复杂度，用最小可行方案
    ↓（仍失败）
3. 人工介入：暂停并请求用户明确指示
```

#### 错误恢复机制

- **每次关键操作前保存状态**：支持回滚到上一版本
- **冲突自动检测**：设定冲突、时间线冲突、人物行为冲突
- **渐进式产出**：即使中断，已完成部分也可独立使用

## Agent团队介绍

| Agent | 职责 | 层级 |
|-------|------|------|
| Creative Director | 把握全局方向，解决核心冲突 | 战略层 |
| Editorial Director | 质量把关，终审校对 | 质量层 |
| World Architect | 世界观与设定构建 | 设计层 |
| Character Lead | 人物塑造与弧光设计 | 设计层 |
| Plot Architect | 情节结构与节奏把控 | 设计层 |
| Scene Writer | 场景描写与叙事执行 | 执行层 |
| Dialogue Specialist | 对话创作与潜台词设计 | 执行层 |
| Prose Polisher | 文笔润色与风格统一 | 执行层 |
| Researcher | 资料考证与真实感增强 | 支持层 |
| Continuity Tracker | 世界观一致性维护 | 支持层 |
